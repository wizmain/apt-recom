# Antigravity Workspace Rules (uv + root-level projects)

## 0. 목적
이 워크스페이스는 "프로젝트별 폴더 독립 구조"를 사용합니다.
각 프로젝트는 워크스페이스 루트에 직접 생성하며, 가상환경은 루트 `.venv` 하나만 사용합니다.

---

## 1) 필수 디렉터리 구조 (Workspace Root)

워크스페이스 구조 예시:

- /.venv
- /<project_name_1>
  - /src
  - /data
  - /images
  - /docs
- /<project_name_2>
  - /src
  - /data
  - /images
  - /docs

규칙:
- `/projects` 폴더는 사용하지 않습니다.
- 프로젝트는 루트에 바로 생성합니다.
- 프로젝트 폴더명은 소문자 스네이크 케이스를 권장합니다.
  - 예: `yes24_scraper`, `oliveyoung_eda`

---

## 2) 가상환경 규칙 (uv + 단일 .venv)
- 가상환경은 워크스페이스 루트의 `.venv`만 사용합니다.
- `.venv`가 존재하면 절대로 새로 생성하지 않습니다.
- 프로젝트 내부에 `.venv`를 만들지 않습니다.
- uv 실행 및 패키지 관리는 항상 루트 기준으로 수행합니다.
- 실행 전 반드시 루트 `.venv` 사용 여부를 확인합니다.

---

## 3) 프로젝트 내부 구조 규칙

각 프로젝트는 아래 구조를 반드시 유지합니다.
/<project_name>/
/src
/data
/images
/docs

### 3-1) 코드 파일
- 코드 파일은 반드시 `/<project_name>/src/` 아래에만 저장합니다.
- 코드 생성 시 docstring 반드시 생성합니다.
- 중요 로직에 주석 생성합니다.
- 루트 또는 다른 프로젝트에 코드 파일을 생성하지 않습니다.
- 파일명은 목적별로 자유롭게 사용합니다.
  - 예: `collect.py`, `clean.py`, `analyze.py`, `report.py`
- `.ipynb` 사용 금지

### 3-2) 데이터 파일
- 모든 데이터 파일은 `/<project_name>/data/` 아래에 저장합니다.
- 필요 시:
  - `data/raw`
  - `data/processed`
  하위 구조를 사용합니다.
- 데이터 파일을 `src` 또는 루트에 저장하지 않습니다.

### 3-3) 시각화 파일
- 시각화 결과는 반드시 `/<project_name>/images/`에 저장합니다.
- 필요 시:
  - `images/eda`
  - `images/report`
  하위 구조 사용 가능
- 이미지 파일을 `src` 또는 `data`에 저장하지 않습니다.

### 3-4) 문서
- 실행 방법, 분석 설명, 결과 문서는 `/<project_name>/docs/`에 저장합니다.
- 간단 설명만 있을 경우 프로젝트 루트 `README.md` 허용

---

## 4) 작업 절차 규칙 (중요)

### 절대 규칙: 코드 작성 전에 계획 먼저

작업 순서:

1) 계획 수립
- 목표
- 입력 데이터
- 출력 결과
- 폴더 구조
- 작업 단계

2) 사용자 확인 질문
- 프로젝트명
- 데이터 위치
- 예상 산출물
- 작업 범위

3) 사용자 승인 이후 코드 작성

사용자 확인 전:
- 코드 작성 금지
- 파일 생성 금지

---

## 5) 작업 시작 체크리스트
작업 시작 시 항상 확인:

- 현재 대상 프로젝트:
  - `/<project_name>/`
- 저장 위치 확인:
  - 코드 → `/src`
  - 데이터 → `/data`
  - 이미지 → `/images`
  - 문서 → `/docs`
- 루트 `.venv` 사용 여부 확인
- 새 `.venv` 생성 시도 금지
- 계획 → 확인 → 코드 순서 준수

---

## 6) 금지 사항

- `/projects` 폴더 생성 금지
- 프로젝트 외부 파일 저장 금지
- 프로젝트별 가상환경 생성 금지
- `.ipynb` 생성 금지
- 데이터와 이미지 위치 혼용 금지
- 사용자 확인 없이 코드 작성 금지

---

## 7) 사용자 확인 템플릿

작업 시작 전 아래 정보를 확인합니다:

- 프로젝트 이름: `<project_name>`
- 데이터 위치: `/data/...`
- 산출물:
  - 데이터: `/data/...`
  - 시각화: `/images/...`
  - 문서: `/docs/...`
- 가상환경: 루트 `.venv` 재사용 (편집됨)
