# 거래-아파트 매핑 개선 계획

## 현재 문제

| 항목 | 값 |
|------|-----|
| 매매 고유단지 | 17,492 |
| 전월세 고유단지 | 18,875 |
| 매핑됨 | 7,972 (45.6%) |
| **미매칭** | **~11,000 (54.4%)** |

### 원인 분석

**1. apt_seq 형태 변경**
- 기존 Dev API: `11110-2445` (고유 시퀀스)
- 2016 수집: `11110_(1-102)` (sggCd + 아파트명 합성) → 아파트명에 특수문자/괄호 포함

**2. 이름 불일치 패턴**
| 거래 데이터 | 마스터 | 원인 |
|-----------|--------|------|
| 에스케이북한산시티 | 에스케이북한산시티**아파트** | "아파트" 접미사 |
| 주은청설 | 주은청설**아파트** | "아파트" 접미사 |
| 중계그린1단지 | 중계그린**아파트** | "단지"→"아파트" |
| 큰마을대림,현대아파트 | 큰마을현대아파트 | 콤마+복합명 |
| 샘터마을2 | 샘터마을 | 숫자 접미사 |
| 황골마을주공1 | (없음) | 마스터에 없음 |

**3. 기존 Dev API 데이터 미활용**
- `apt_trade_total_2023_2026.csv`에 aptSeq(고유키) + bonbun/bubun/umdCd(주소정보) 포함
- 이 데이터로 aptSeq→PNU 매핑을 먼저 구축하면, 2016 데이터의 아파트명→aptSeq→PNU 체인 가능

---

## 개선 계획 (4단계)

### Step 1: Dev API aptSeq 기반 매핑 복원
기존 `apt_trade_total_2023_2026.csv`의 aptSeq로 정확한 매핑 구축:
```
apt_trade_total_2023_2026.csv (aptSeq 포함)
  → aptSeq별 (sggCd, aptNm, umdNm, bonbun, bubun) 추출
  → apartments 테이블과 매칭:
     방법1: sggCd + 정규화이름
     방법2: bjdCode(sggCd+umdCd) + bonbun/bubun → PNU 직접 생성
  → aptSeq→PNU 매핑 테이블
```
예상: 기존 6,709개 aptSeq 중 85%+ 매핑 가능

### Step 2: aptSeq↔합성키 브릿지
Dev API의 aptSeq와 2016 수집의 합성키를 연결:
```
Dev API:  aptSeq=11110-2445, aptNm=경희궁자이(2단지), sggCd=11110
2016:     apt_seq=11110_경희궁자이(2단지), aptNm=경희궁자이(2단지), sggCd=11110

→ sggCd + aptNm으로 조인하면 합성키→aptSeq→PNU 체인 완성
```

### Step 3: 이름 매칭 개선
미매칭 패턴별 규칙 추가:
```python
# 1. "아파트" 접미사 추가 매칭
trade_nm + "아파트" → master_nm

# 2. 숫자/단지 접미사 제거
"중계그린1단지" → "중계그린" → "중계그린아파트"
"샘터마을2" → "샘터마을"

# 3. 콤마/특수문자 분리
"큰마을대림,현대아파트" → ["큰마을대림", "현대아파트"] → 각각 매칭

# 4. 공백 변형
"에스케이 북한산시티" vs "에스케이북한산시티아파트" → 공백 무시 매칭
```

### Step 4: 가격 점수 재계산
매핑 개선 후 apt_price_score 재계산:
- 매핑된 거래 데이터 기반 ㎡당 가격
- 시군구 평균 대비 점수
- 전세가율
