# 아파트 주변 안전점수 산출 모델 v2

> 작성일: 2026-04-05
> 목적: 아파트 주변 안전도를 실서비스 기준으로 정량화하기 위한 v2 점수 모델 설계

---

## 1. 문서 목적

본 문서는 아파트 주변 안전도를 평가하기 위한 **실서비스용 안전점수 v2 모델**을 정의한다.

v1 리서치 모델은 데이터 소스와 평가 항목을 폭넓게 정리하는 데 초점을 두었지만, 실제 서비스 적용 관점에서는 아래 문제가 있었다.

- 정부 공식 지수와 세부 데이터를 함께 더해 같은 위험을 중복 반영할 가능성
- 시군구 단위 데이터와 반경 기반 데이터를 같은 층위에서 직접 합산하는 문제
- 극단값에 취약한 정규화 방식
- 좌표 누락, 최신성 차이, 지역별 커버리지 차이를 반영하지 못하는 문제
- 법적·윤리적 제약이 있는 데이터를 메인 점수에 직접 포함한 문제

v2는 위 문제를 보완해 **설명 가능성**, **운영 가능성**, **데이터 신뢰성**을 함께 고려하는 모델로 재구성한다.

---

## 2. v2 설계 원칙

### 2.1 중복 반영 방지

- `행정안전부 지역안전지수`는 범죄, 교통사고, 화재 등 여러 지표를 이미 반영한 공식 지수다.
- 따라서 범죄, 교통, 소방 데이터를 별도로 계산하는 서비스 모델에서는 지역안전지수를 높은 비중으로 직접 합산하지 않는다.
- v2에서는 지역안전지수를 **광역 보정 지표** 또는 **참고 지표**로만 사용한다.

### 2.2 공간 해상도 분리

서로 다른 공간 단위의 데이터는 같은 층위에서 직접 합산하지 않는다.

- `미시권역`: 아파트 기준 반경 400m~800m
- `접근성`: 이동시간 5~15분권
- `광역권역`: 시군구 단위
- `단지 내부`: 공동주택 단지 단위

각 층위별 점수를 먼저 계산한 뒤, 최종 단계에서만 통합한다.

### 2.3 신뢰도 병행 표기

최종 안전점수와 별도로 `데이터 신뢰도 점수`를 함께 제공한다.

이유:

- 일부 지자체 데이터는 최신이지만 일부는 오래되었을 수 있다.
- 좌표 기반 데이터라도 주소 정확도, 좌표계, 누락 여부가 다를 수 있다.
- 사용자는 점수뿐 아니라 점수의 **신뢰 수준**도 알아야 한다.

### 2.4 법적·윤리적 민감 데이터 최소화

- 개별 성범죄자 위치 정보처럼 대량 수집 및 정밀 위치화에 제약이 있는 데이터는 메인 점수에서 제외하거나 참고 지표로만 사용한다.
- 서비스 품질보다 법적·윤리적 안정성을 우선한다.

## 3. 최종 평가 구조

안전점수는 내부적으로 0~100 표준점수로 계산하고, 사용자에게 100점 만점으로 노출한다.

```text
최종안전점수 =
  미시환경점수 55%
+ 접근성점수 20%
+ 광역위험점수 15%
+ 단지내부점수 10%
```

별도 제공:

- 데이터신뢰도점수: 0~100
- 정부공식안전등급: 참고 정보

---

## 4. 평가 영역 정의

### 4.1 미시환경점수 (55점)

아파트 주변 반경 기반의 체감 안전 환경을 평가한다.

구성:

- 범죄주의구간 / 생활안전지도 등급
- CCTV 밀도
- 야간조명 환경
- 보행자 중심 교통 위험
- 반경 내 치안안전시설 밀도

권장 배점:

| 세부 항목 | 배점 |
|---|---:|
| 생활안전지도 범죄주의구간 | 20 |
| CCTV 밀도 | 12 |
| 야간조명 | 10 |
| 보행/어린이 교통안전 | 13 |

### 4.2 접근성점수 (20점)

사건·사고·응급상황 시 공공 인프라에 얼마나 빠르게 접근 가능한지 평가한다.

구성:

- 경찰 접근성
- 소방 접근성
- 응급의료 접근성

권장 배점:

| 세부 항목 | 배점 |
|---|---:|
| 경찰 | 6 |
| 소방 | 7 |
| 응급의료 | 7 |

### 4.3 광역위험점수 (15점)

시군구 단위의 구조적 위험 수준을 평가한다.

구성:

- 시군구 범죄율
- 시군구 교통사고율
- 자연재해 위험
- 행정안전부 지역안전지수 보정값

권장 배점:

| 세부 항목 | 배점 |
|---|---:|
| 시군구 범죄율 | 6 |
| 시군구 교통사고율 | 4 |
| 자연재해 위험 | 3 |
| 지역안전지수 보정 | 2 |

### 4.4 단지내부점수 (10점)

단지 안쪽의 물리적 안전성과 관리 수준을 평가한다.

구성:

- 단지 CCTV 대수
- 경비관리 인력
- 출입통제 및 관리 방식
- 소방·방재 설비 관련 기초 정보

권장 배점:

| 세부 항목 | 배점 |
|---|---:|
| 공동주택 내부 안전성 | 10 |

## 5. 메인 점수에서 제외 또는 축소할 항목

### 5.1 성범죄자 거주 정보

권장 정책:

- 메인 점수에서 제외
- 필요 시 별도 참고 지표로 분리

이유:

- 개별 조회 중심 구조라 대량 수집에 제약이 있다.
- 위치 정밀도와 공개 범위에 한계가 있다.
- 점수 설명 책임이 크고 오해 소지가 높다.

### 5.2 지역안전지수 직접 합산

권장 정책:

- 최종 점수의 큰 비중으로 직접 합산하지 않음
- 광역위험 보정용 소규모 항목으로만 사용

이유:

- 범죄, 교통, 소방, 의료 등과 내용이 중복될 수 있다.

## 6. 정규화 방식

v1의 `1 - (해당값 / 전국최고값)` 방식은 극단값에 취약하므로 사용하지 않는다.

### 6.1 권장 방식

우선순위:

1. percentile rank 기반 점수화
2. winsorized z-score
3. 로그 변환 후 min-max

### 6.2 실무 권장 기본안

- 상위 95퍼센타일을 초과하는 값은 95퍼센타일로 캡
- 필요 시 로그 변환
- 이후 0~100 구간으로 정규화

예시:

```text
위험지표정규값 = normalize(log(clip(metric, p5, p95)))
안전점수 = 100 - 위험지표정규값
```

장점:

- 특정 지역의 이상치가 전체 점수를 왜곡하는 문제를 줄일 수 있다.
- 범죄, 교통사고 같은 long-tail 분포에 더 적합하다.

## 7. 거리 계산 원칙

직선거리만으로 안전 인프라 접근성을 계산하지 않는다.

### 7.1 항목별 계산 방식

- 경찰/소방/응급의료: `자동차 이동시간` 우선
- 야간 도보 환경: `보행 네트워크 거리` 또는 도보시간 우선
- 반경 기반 시설 밀도: 직선거리 사용 가능

### 7.2 구현 단계별 정책

- MVP: Haversine + 도심/비도심 보정계수
- 운영 고도화: 카카오맵 또는 네이버 길찾기 기반 이동시간 적용

## 8. 데이터 신뢰도 점수

최종 점수와 별도로 아래 공식을 사용해 신뢰도 점수를 산출한다.

```text
데이터신뢰도점수 =
  최신성 35
+ 완전성 25
+ 좌표정확도 20
+ 지역커버리지 20
```

### 8.1 평가 기준

| 항목 | 설명 |
|---|---|
| 최신성 | 최근 갱신일 기준. 오래된 자료일수록 감점 |
| 완전성 | 필수 필드 누락률, 주소/좌표/분류값 존재 여부 |
| 좌표정확도 | 좌표계 명확성, 지오코딩 성공률, 위치 오차 수준 |
| 지역커버리지 | 전국 커버 여부, 특정 지자체 누락 여부 |

### 8.2 사용자 노출 예시

- 안전점수: 82점
- 안전등급: A
- 데이터신뢰도: 68점
- 해석: 조명 데이터 커버리지가 낮아 야간 안전 평가는 보수적으로 계산됨

## 11. v2 영역별 계산 개요

### 11.1 미시환경점수

```text
미시환경점수 =
  범죄주의구간점수 * 0.36
+ CCTV점수 * 0.22
+ 야간조명점수 * 0.18
+ 보행교통안전점수 * 0.24
```

### 11.2 접근성점수

```text
접근성점수 =
  경찰접근성 * 0.30
+ 소방접근성 * 0.35
+ 응급의료접근성 * 0.35
```

### 11.3 광역위험점수

```text
광역위험점수 =
  범죄율안전점수 * 0.40
+ 교통사고안전점수 * 0.27
+ 자연재해점수 * 0.20
+ 지역안전지수보정 * 0.13
```

### 11.4 단지내부점수

```text
단지내부점수 =
  단지CCTV * 0.40
```

## 14. 운영 정책

- 점수는 절대적인 안전 보장이 아니라 상대적 비교 지표로 제공한다.
- 데이터 최신성 차이가 큰 경우 사용자에게 경고 문구를 노출한다.
- 법적 민감 데이터는 참고 정보로만 취급한다.
- 데이터가 부족한 지역은 점수를 억지로 계산하지 않고 신뢰도 하락으로 표시한다.
- 새 지표를 추가할 때는 기존 점수와의 중복 반영 여부를 먼저 검토한다.

## 15. 결론

안전점수 v2는 단순 합산형 리서치 모델에서 벗어나, 실서비스 운영에 필요한 다음 요소를 반영한 구조다.

- 미시환경과 광역위험의 분리
- 정부 공식 지수의 중복 반영 방지
- 단지 내부 안전성 반영
- 데이터 신뢰도 병행 제공
- 법적·윤리적 리스크 최소화

향후 구현은 `MVP 점수 계산 → 신뢰도 점수 병행 → 이동시간/재해/CPTED 고도화` 순서로 진행하는 것을 권장한다.
