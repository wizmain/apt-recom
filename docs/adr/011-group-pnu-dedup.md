# ADR-011: 동일 단지 중복 PNU 통합 (group_pnu)

- **상태**: Accepted
- **날짜**: 2026-03-29

## 맥락

건축물대장 기준으로 하나의 아파트 단지(예: 은평뉴타운 상림마을)가 동별로 별도 PNU를 부여받아, apartments 테이블에 같은 단지명으로 여러 행이 존재했다. 스코어링 시 동일 단지가 1위와 3위를 동시에 차지하는 문제가 발생했다.

- 349개 그룹, 1,107건 관련 (전체 16,755의 6.6%)
- 통합 대상: 758건 (제거 가능한 하위 PNU)

## 결정

`apartments` 테이블에 `group_pnu TEXT` 컬럼을 추가하여 동일 단지를 그룹핑한다.

- 단독 아파트: `group_pnu = pnu` (자기 자신)
- 중복 그룹: 세대수 최대인 PNU를 대표(`group_pnu = pnu`)로 지정, 나머지는 `group_pnu = 대표PNU`
- 대표 PNU의 `total_hhld_cnt`에 그룹 전체 세대수를 합산

## 그룹핑 기준

- **동일 단지명** (`bld_nm`)
- **근접 좌표** (`ROUND(lat, 3)`, `ROUND(lng, 3)` — 약 100m 이내)

## 적용 범위

- 목록/검색/스코어링 쿼리: `WHERE group_pnu = pnu` 조건 추가
- 상세 조회: PNU 직접 지정이므로 영향 없음 (하위 PNU도 상세 조회 가능)
- 의존 테이블(apt_facility_summary, school_zones 등): 기존 데이터 유지, 삭제하지 않음

## 근거

- **데이터 레벨 해결**: 스코어링 API에서 임시 중복제거하는 것보다 근본적
- **기존 데이터 보존**: 행을 삭제하지 않고 컬럼으로 관리하므로 롤백 용이
- **최소 스키마 변경**: 컬럼 1개 추가 + 쿼리에 조건 1줄 추가

## 결과

- 지도 마커, 스코어링, 검색 결과에서 동일 단지 중복 제거
- 대표 PNU에 합산 세대수 반영 (예: 은평뉴타운 상림마을 450세대)
- 수정 파일: `database.py`, `apartments.py`, `nudge.py`, `tools.py`
