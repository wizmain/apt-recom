# ADR-009: 뷰포트 기반 마커 로딩

- **상태**: Accepted
- **날짜**: 2026-03-21

## 맥락

수도권 아파트 16,755개를 지도에 한 번에 표시하면 초기 로딩이 느리고 렌더링 성능이 저하된다.

## 결정

카카오 지도의 현재 뷰포트(bounds) 영역에 해당하는 아파트만 API에서 반환한다. 지도 이동/줌 시 새로운 bounds로 재요청한다.

## 근거

- **성능**: 전체 16,755개 → 뷰포트 내 수백 개만 로딩하여 초기 로딩 99.5% 감소.
- **서버 부하**: 매번 전체 데이터를 전송하지 않으므로 네트워크/서버 부하가 줄어든다.
- **DB 최적화**: `apartments` 테이블의 `lat`, `lng` 컬럼으로 bounds 필터링이 가능하다.

## API

```
GET /api/apartments?sw_lat=37.4&sw_lng=126.8&ne_lat=37.7&ne_lng=127.2
```

## 트레이드오프

- 지도 이동할 때마다 API 호출이 발생한다 (debounce로 완화).
- 넛지 스코어링도 bounds 기반이므로, 뷰포트 밖의 고득점 아파트는 보이지 않는다.

## 결과

- `apartments.py` 라우터에서 bounds 파라미터로 WHERE 절 필터링.
- 프론트엔드에서 지도 `idle` 이벤트마다 현재 bounds를 전달하여 재요청.
- 넛지 스코어링 API도 동일한 bounds 필터를 지원한다.
