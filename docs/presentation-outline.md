# 아파트 추천 서비스 개발 스토리

---

## 1. 왜 이 서비스를 만들었는가

- **문제 인식**: 아파트를 고를 때 가격, 교통, 학군, 안전 등 고려할 요소가 너무 많다
- **기존 서비스의 한계**: 네이버 부동산, 직방 등은 매물 중심 — "나의 라이프스타일에 맞는 추천"은 없다
- **핵심 질문**: "나한테 맞는 아파트"를 데이터로 찾을 수 있을까?
- **목표**: 9가지 라이프스타일(가성비, 출퇴근, 학군, 안전, 투자, 반려동물, 신혼, 자연친화, 시니어) 기반으로 점수를 매겨 추천하는 서비스

---

## 2. 데이터 수집 — 공공데이터의 바다에서 건져올리기

- **아파트 마스터**: 건축물대장 + K-APT → 수도권 16,755개 아파트
- **15종 시설 데이터**: 병원, 학교, 공원, 지하철, 버스, 도서관, 경찰서, CCTV, 편의점, 약국, 마트, 유치원, 동물병원, 반려동물시설, 소방서 → 182,902건
- **거래 데이터**: 매매 + 전월세 10년치(2016~2026) → 787만건
- **학군 데이터**: 초/중/고 통학구역 Shapefile → GIS 공간 조인
- **인구 데이터**: KOSIS 읍면동별 인구통계 → 1,192개 읍면동
- **범죄 데이터**: 경찰청 5대범죄 시군구별 통계 + 유동인구 보정
- **어려웠던 점**: API마다 다른 포맷, 좌표 오류 수정, 인천 지오코딩 실패 → 범위 검증 로직 추가

## 3. EDA — 데이터가 말해준 것들

### 연령대별 시설 선호도 분석
- **발견 1**: 연령대별로 시설 선호도가 완전히 뒤집힌다
  - 20대: 반려동물시설 선호도 최고 (r=0.69) — 반려동물은 20대의 문화
  - 40대: 병원 접근성 중시 (r=0.58) — 가족 건강이 우선
  - 60대: 학교 회피 (r=-0.76), 병원 거리 선호 (r=+0.82) — 조용한 환경 추구
- **의미**: "하나의 추천 공식"은 불가능 → 라이프스타일별로 가중치를 완전히 다르게 설계해야 함

### 연령대별 가격 패턴 분석
- **발견 2**: 40대가 프리미엄 시장을 지배한다 (r=0.791)
  - 40대 최상위 가격대(Q4) 구매 비중 33% — 학군 프리미엄을 기꺼이 지불
  - 20대는 강하게 가성비 추구 (r=-0.622)
  - 50대부터 가치 지향으로 전환 (r=-0.476)
- **의미**: 가성비 넛지(20~30대) vs 학군 넛지(40대)의 가중치 설계 근거

### 학군 × 가격 분석
- **발견 3**: 학군이 부동산 가격의 최강 변수
  - 강남반원초 학구역: ㎡당 2,910만원 vs 신북초: ㎡당 118만원 → **24.7배 차이**
  - 서울 평균 1,050만/㎡ vs 경기 530만 vs 인천 370만 → 약 3배
- **의미**: 학군 점수는 단순 거리가 아닌 "학군 프리미엄"이 반영되어야 함

### 교통 인프라 분석
- **발견 4**: 서울은 지하철, 경기는 버스
  - 서울: 409개 지하철역, 환승역 31.8% — 노선 다양성이 핵심
  - 경기 외곽: 38,089개 버스정류장이 대안 — 지하철 부재를 보완
- **의미**: 출퇴근 넛지에서 지하철 가중치 45% + 버스 35%의 근거

### CCTV × 안전 분석
- **발견 5**: CCTV 72.2%가 "생활방범" 용도 — 범죄 특화가 아님
  - 2019년 설치 피크 (3,787건) — 사회적 안전 인식 변화
  - 도시 밀집 지역에 집중 → 외곽은 CCTV 인프라 부족
- **의미**: CCTV 밀도만으로는 안전 점수 불충분 → 범죄율 데이터 추가 필요 (이후 수집)

## 4. 점수 설계 — EDA에서 공식으로

### 라이프 점수 시스템
- **원칙**: 각 아파트에서 주변 시설까지의 거리를 점수화 (가까울수록 100점, 최대 거리 초과 시 0점)
- **기술**: BallTree 공간 인덱스로 5km 반경 시설 매핑 → **4,900만 매핑 행**

### 9가지 라이프 항목과 EDA 기반 가중치
| 항목 | 핵심 요소 | 가중치 근거 (EDA 인사이트) |
|------|----------|--------------------------|
| 가성비 | 가격 30% + 전세가율 15% + 지하철 15% | 20대 가성비 선호 (r=-0.622) |
| 출퇴근 | 지하철 45% + 버스 35% + 편의점 20% | 서울 지하철 중심, 경기 버스 보완 |
| 학군 | 학교 30% + 유치원 25% + 도서관 25% | 학군 프리미엄 24.7배 차이 |
| 안전 | 범죄율 25% + CCTV 25% + 경찰서 15% | CCTV만으로 불충분 → 범죄율 보강 |
| 투자 | 가격 25% + 전세가율 20% + 지하철 20% | 40대 프리미엄 수용 (r=0.791) |
| 반려동물 | 동물병원 30% + 반려시설 35% + 공원 35% | 20대 반려동물 선호 (r=0.69) |
| 신혼육아 | 유치원 20% + 학교 20% + 공원 15% | 30대 병원 접근 중시 (r=-0.74) |
| 시니어 | 병원 25% + 약국 20% + 편의점 20% | 60대 조용한 환경 + 의료 접근 |
| 자연친화 | 공원 50% + 도서관 25% + 반려시설 25% | 공원 선호도 전 연령대 높음 |

### 사용자 커스텀
- 항목별 가중치를 슬라이더로 직접 조절 → 나만의 점수 공식 생성

## 5. 안전 점수 — 범죄 데이터와의 씨름

### 4단계 진화 과정
- **1차**: CCTV 밀도만으로 안전 점수 → CCTV 72%가 생활방범 용도, 범죄와 직접 연관 부족
- **2차**: 경찰청 범죄통계 → 경찰청 단위(21개)만 제공, 서울 전체가 하나
- **3차**: 대검찰청 KOSIS 데이터 → 시군구 54개 확보, 경기도 없음
- **4차**: data.go.kr 경찰서별 5대범죄 → **전국 230개 시군구** 완전 확보

### 유동인구 보정
- **문제 발견**: 종로/중구/강남이 "가장 위험"으로 산출 — 주민등록인구 기준 왜곡
  - 중구: 주민 11.8만명 vs 일 유동인구 50만+ → 범죄율 4배 과대 산정
- **해결**: 서울시 생활인구 데이터로 유동인구 보정계수 적용
  - 중구 4.2배, 종로 3.8배, 강남 2.2배
- **결과**: 중구 0점→72.5점, 종로 24점→82.5점으로 현실적인 점수

### 범죄 유형 분석
- 살인/강도/강간·강제추행/절도/폭력 5대범죄 비율 시각화
- 종로/중구: 절도·폭력이 대부분 (관광지·유흥가 집중) → 주거지는 상대적 안전
- 안전 탭에서 인포그래픽으로 범죄 유형별 비율 + 유동인구 보정 설명 제공

## 6. AI 챗봇 — 대화로 아파트를 추천하다

- **LLM 연동**: OpenAI GPT-4o 기반 (Claude, Gemini 전환 가능)
- **Tool Calling**: 7개 도구
  - 아파트 검색 / 상세조회 / 비교 / 시세 분석 / 학군 조회 / 지식검색(RAG) / 출퇴근 조회
- **SSE 스트리밍**: 글자 단위 실시간 응답 (도구 실행 상태 → 답변 생성 중 → 텍스트 스트리밍)
- **지도 양방향 연동**:
  - 지도에서 아파트 클릭 → 챗봇에 PNU + 이름 전달 → 정확한 조회
  - 챗봇 검색 결과 → 지도에 마커 표시 + 자동 포커싱
- **출퇴근 조회**: ODSay 대중교통 API → "강남역까지 몇 분?" 자연어 질의
- **안전 점수 설명**: 종합/CCTV/범죄 점수 구분, 유동인구 보정 근거, 범죄 유형 분석

## 7. 프론트엔드 — 지도 위에서 아파트를 탐색하다

- **기술 스택**: React 19 + Vite + TailwindCSS + Kakao Maps
- **핵심 UX**:
  - 라이프 항목 선택 → 실시간 순위 마커 표시 (1위 빨강~5위 로즈)
  - 마커 클릭 → 팝업 (상세보기 / 챗봇 분석 / 비교담기)
  - 키워드 검색 → 지도 자동 이동
  - 가중치 슬라이더 → 점수 즉시 재계산
- **상세 모달**: 6개 탭 (기본정보, 가격분석, 주변시설, 학군, 안전, 인구)
  - 가격 추이 차트, 라이프 점수 레이더 차트, 인구 피라미드
  - 안전 탭: 링 게이지 3종 + 범죄 유형 바 차트 + CCTV 밀도 게이지
- **비교 기능**: 2개 아파트 선택 → 좌우 분할 비교 (점수 VS 바, 시설 거리, 학군, 안전)

## 8. 피드백 루프 — 챗봇 답변을 계속 개선하기

- **피드백 수집**: 각 AI 답변에 👍/👎 버튼 + 6가지 태그 (정보 부정확, 너무 김, 원하는 답 아님, 점수 이상, 정보 부족, 가독성)
- **데이터 저장**: 사용자 질문 + AI 답변 + 사용 도구 + 평점 + 태그 → PostgreSQL
- **향후 계획**:
  - 피드백 대시보드: 만족도 추이, 불만 태그 분포
  - Few-shot 예시: 좋은 평가 받은 답변을 시스템 프롬프트에 예시로 주입
  - A/B 프롬프트 테스트

## 9. 데이터 품질과 호스팅

### 좌표 오류와의 전쟁
- **인천 좌표 오류 발견**: "부평신일해피트리더루츠" 좌표가 경기 성남으로 찍힘
- **원인**: 지오코딩 시 아파트명만으로 Kakao 검색 → 타 지역 동명 아파트 좌표 반환 (153건)
- **해결**: `_is_incheon()` 범위 검증 + "인천광역시" 접두어 강제 → 0건
- **교훈**: 지오코딩은 반드시 행정구역 범위 검증 필요

### 로컬에서 클라우드로
- **DB 마이그레이션**: SQLite(8.3GB) → PostgreSQL (4,900만 행 시설 매핑 포함)
- **아키텍처**: Railway(Backend + PostgreSQL) + Cloudflare Pages(Frontend)
- **비용**: Railway ~$8/월 + Cloudflare Pages 무료 = **월 $8**

## 10. 앞으로의 방향

- **데이터 고도화**: 생활안전지도 격자 밀도 데이터, 소음/일조량 데이터
- **ML 모델**: 가격 예측(XGBoost), 유사 아파트 추천(KNN), 투자 가치 예측
- **사용자 기능**: 관심 아파트 저장, 가격 알림, 맞춤 리포트 PDF
- **챗봇 고도화**: 피드백 기반 프롬프트 자동 개선, 대화 맥락 기억
- **모바일**: 반응형 UI + PWA
- **오픈소스**: GitHub 공개, 다른 도시(부산/대전 등)로 확장 가능 구조
