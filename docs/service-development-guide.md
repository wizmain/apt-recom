# 집토리 — 라이프스타일 아파트 찾기 서비스 구축 가이드

> 서울+경기+인천 수도권 아파트 데이터를 수집·분석하여 라이프스타일 기반 아파트 추천 서비스를 구축하는 전체 과정을 정리합니다.

---

## 서비스 개요

"집토리"는 수도권 16,755개 아파트를 대상으로, 사용자의 라이프스타일에 맞는 아파트를 데이터 기반으로 추천하는 서비스입니다. 기존 부동산 서비스가 매물 중심으로 정보를 제공하는 것과 달리, 가성비·출퇴근·학군·안전·투자·반려동물·신혼육아·시니어·자연친화 9가지 라이프스타일 항목별로 점수를 산정하여 추천합니다.

서비스 구축을 위해 공공데이터포털, KOSIS, 서울열린데이터 등에서 아파트 기본정보, 15종 주변시설(182,902건), 10년치 실거래 787만건, 학군 Shapefile, 인구통계, 범죄통계를 수집했습니다. 수집된 데이터는 BallTree 공간 인덱스로 아파트-시설 간 거리를 계산(4,900만 매핑)하고, 거래 데이터를 아파트에 매핑(99.6%)하며, GIS 공간조인으로 학군을 배정하는 과정을 거쳐 서비스 데이터로 가공됩니다.

프론트엔드는 카카오 지도 위에서 아파트를 탐색하고 필터링하는 UI를 제공하며, AI 챗봇 "집토리"는 자연어로 아파트 분석, 비교, 출퇴근 시간 조회를 지원합니다. 백엔드는 FastAPI + PostgreSQL로 구성되며, Railway와 Cloudflare Pages에 월 $8 비용으로 호스팅됩니다.

## 구축 과정 요약

### 자료 수집
수도권 아파트 서비스에 필요한 데이터를 6개 분류로 수집했습니다. 아파트 기본정보는 건축물대장 API와 K-APT에서 16,755개 단지의 세대수·층수·준공일을 확보하고, Vworld와 카카오 지오코딩으로 좌표를 매핑했습니다. 주변시설은 data.go.kr, 소상공인 상가업소, KRIC 등 10여 개 출처에서 병원·학교·공원·지하철·버스·편의점 등 15종 182,902건을 수집했습니다. 거래 데이터는 국토교통부 실거래가 API에서 2016~2026년 매매 236만건, 전월세 551만건을 포함한 787만건을 확보했습니다. 학군은 교육부 Shapefile 4종(초/중/고/교육행정구역)을 활용했고, 인구는 KOSIS에서 시군구별·읍면동별 통계를, 범죄는 경찰청·대검찰청 5대범죄 데이터를 시군구 단위로 수집했습니다.

### 데이터 연결 및 가공
수집한 원시 데이터를 서비스에 활용할 수 있도록 5단계 가공을 거칩니다. 첫째, 아파트와 시설 간 거리를 BallTree 공간 인덱스로 5km 반경 내 매핑하여 4,900만 행의 거리 테이블을 생성합니다. 둘째, 거래 데이터의 아파트 식별자를 4단계 매칭(이름→퍼지→주소→지번)으로 99.6% 연결합니다. 셋째, 아파트 좌표를 GeoPandas로 학군 Shapefile과 공간 조인하여 초/중/고 배정학교를 매핑합니다. 넷째, 시설 거리를 0~100 점수로 변환하고 라이프스타일 항목별 가중치를 적용하여 9가지 라이프 점수를 산정합니다. 다섯째, 범죄율에 유동인구 보정계수를 적용하고, CCTV 밀도와 결합하여 종합 안전 점수를 계산합니다.

### 서비스 구축
백엔드는 FastAPI + PostgreSQL로 11개 REST API를 구성했습니다. 아파트 목록은 지도 뷰포트 기반으로 해당 영역만 반환하여 초기 로딩을 99.5% 줄였고, 챗봇은 SSE 스트리밍으로 실시간 응답합니다. 프론트엔드는 React + Vite + TailwindCSS에 카카오 지도를 연동하여, 라이프 항목 선택 → 순위 마커 → 상세 모달(6탭) → 아파트 비교 → AI 챗봇 분석의 탐색 흐름을 제공합니다. AI 챗봇 "집토리"는 OpenAI GPT-4o 기반 7개 Tool(검색·상세·비교·시세·학군·지식·출퇴근)을 갖추며, 사용자 피드백을 수집하여 답변 품질을 개선합니다. 배포는 Railway(백엔드+DB)와 Cloudflare Pages(프론트)로 월 $8에 운영됩니다.

## 2. 데이터 연결 및 가공 과정

### 2.1 아파트-시설 거리 매핑

```
아파트 16,755개 × 시설 182,902개
    ↓ BallTree (haversine) 5km 반경 매핑
아파트-시설 매핑 테이블: 49,085,118행
    ↓ 시설유형별 집계
아파트-시설 요약 테이블: 131,139행
  (아파트별 × 시설유형별 최근접 거리, 1km/3km/5km 내 개수)
```

### 2.2 거래-아파트 매핑

```
거래 데이터의 apt_seq (아파트 식별자)
    ↓ 4단계 매칭
    ├ 1차: 정확 이름 매칭 (39.8%)
    ├ 2차: 퍼지 매칭 (84.8%)
    ├ 3차: 주소 기반 매칭 (88.1%)
    └ 4차: Dev API aptSeq + 브릿지 + 지번 (99.6%)
    ↓
trade_apt_mapping 테이블: 17,910건 (매매 99.6%, 전월세 94.6%)
```

### 2.3 학군 매핑

```
아파트 좌표 (lat, lng)
    ↓ GeoPandas Spatial Join
    ├ 초등학교 통학구역 Shapefile
    ├ 중학교 학교군 Shapefile
    ├ 고등학교 학교군 Shapefile
    └ 교육행정구역 Shapefile

```

## 3. 서비스 요구사항 및 기술 스택

### 3.1 서비스 요구사항

#### 핵심 기능
- **라이프스타일 기반 아파트 추천**: 9개 항목(가성비, 출퇴근, 학군, 안전, 투자, 반려동물, 신혼, 시니어, 자연) 선택 → 점수 순 추천
- **지도 기반 탐색**: 카카오 지도 위 아파트 마커, 클러스터링, 순위별 색상 핀
- **필터링**: 면적, 가격, 층수, 세대수, 준공연도 5개 필터
- **아파트 상세 정보**: 6탭 — 기본정보, 가격분석, 주변시설, 학군, 안전, 인구
- **아파트 비교**: 2개 아파트 선택 → 좌우 분할 비교 (점수 VS 바, 시설, 학군, 안전)
- **AI 챗봇 (집토리)**: 자연어 아파트 분석, 추천, 비교, 출퇴근 조회
- **출퇴근 시간 조회**: ODSay 대중교통 API 연동
- **피드백 수집**: 챗봇 답변 👍/👎 + 태그 + 자유 코멘트

#### 비기능 요구사항
- 초기 로딩 시간 최소화 (뷰포트 기반 마커 로딩)
- 챗봇 응답 스트리밍 (SSE)
- 모바일 기본 지원 (반응형 레이아웃)
