# 라이프점수(넛지 스코어링) 적합성 진단 및 개선 로드맵

- 작성일: 2026-07-03
- 범위: 넛지 스코어링 전체 파이프라인 (배치 집계 → 런타임 점수 계산 → 검증 체계)
- 방법: 코드 정밀 분석 + 실증 프로브(지역 4곳 × 넛지 9종 점수 분포/중복도) + 국내외 방법론·데이터 소스 웹 리서치

---

## 1. 종합 평가

현행 체계는 **"사전 집계(ADR-004) + 비선형 거리 감쇠 + 밀도 블렌딩 + 넛지별 가중합"** 구조로, 응답 속도와 구현 단순성 관점에서 합리적이다. 넛지 간 차별성도 실측 기준 양호하다(top20 Jaccard 평균 0.05~0.12). 그러나 다음 세 축에서 적합성 한계가 확인됐다.

| 축 | 판정 | 근거 |
|----|------|------|
| **정확성** | ⚠️ 결함 있음 | crime 테이블 참조 버그(§3.1), 시설 결측=0점 처리, 시설 품질 미반영 |
| **공정성(지역)** | ⚠️ 편향 존재 | 도시 밀도가 모든 넛지를 밀어올림 — 강남이 '자연친화'까지 1위(§2) |
| **검증 가능성** | ❌ 부재 | 점수 타당성을 확인하는 자동 검증/피드백 루프 없음, ML 재학습 비자동 |

## 2. 실증 프로브 결과 (2026-07-03, 로컬 DB)

### 2.1 지역별 점수 분포 (top10, sigungu_code 기준)

| 넛지 | 강남구 | 성남분당구 | 청주흥덕구 | 전남무안군 |
|------|:---:|:---:|:---:|:---:|
| education | 85.6~81.9 | 90.5~82.2 | 86.8~85.0 | 75.5~60.9 |
| nature | **93.0**~89.8 | 89.5~84.6 | 88.5~85.8 | 81.0~73.7 |
| commute | 94.4~91.8 | 89.0~81.8 | 75.3~74.0 | 74.0~67.3 |
| senior | 92.7~91.8 | 90.8~88.5 | 93.2~91.5 | 81.6~76.0 |

**발견:**
1. **도시 밀도 편향**: 강남구가 '자연친화'에서도 93.0으로 최고 — nature 가중치가 park .487 + library .257 + pet_facility .256 로 사실상 "도시 시설 밀도" 지표. 산·하천·녹지율 등 실제 자연 요소 부재.
2. **top-N 점수 압축**: 상위 10개가 1~4점 폭에 몰림(강남 senior 92.7~91.8) → 순위의 변별력이 소수점 차이에 의존, 사용자에게 순위 근거 설명력 약함.
3. **지역 간 절대점수 격차**: 군 지역은 전 넛지에서 10~20점 낮음. 지역 내 랭킹은 유효하나, 지역 간 비교(딥링크·탐색 유입) 시 "무안군 1위 60.9 vs 강남 10위 81.9"가 오독될 수 있음.

### 2.2 넛지 간 차별성 (top20 Jaccard)

- 강남구 평균 0.05, 청주흥덕구 평균 0.12 — **차별화 양호**.
- 최고 중복: cost↔investment 0.48(청주) — 둘 다 price_score/jeonse 공유, 예상된 상관.

## 3. 구조적 문제 (우선순위순, file:line 근거)

### 3.1 [버그] safety 넛지의 crime 지표가 구 테이블을 참조
- `routers/nudge.py:275` 는 `sigungu_crime_score`(**77행**, 초기 적재분)를 조회하지만, 2026-05 전국 범죄 데이터 적재(KOSIS A030)는 `sigungu_crime_detail`(**268행**)에 기록됨 (`batch/annual/build_crime_detail.py:238`).
- 영향: score_crime(safety 가중치 **0.233**)이 77개 시군구 외 전 지역에서 결측 → 조건부 0점 처리(nudge.py:283) → **전국 대부분 지역의 safety 점수 구조적 하락**.
- 수정: 참조 테이블 교체(또는 view 통일) — 소규모, 즉시 가능.

### 3.2 시설 결측 = 0점 (중립 아님)
- `scoring.py:259`, `:327` — 데이터 미수집과 실제 부재를 구분하지 않고 0점. subway 만 비수도권 50점 보정 예외(scoring.py:299-305).
- 영향: POI 수집 커버리지가 낮은 비도시 지역의 구조적 저평가(§2.1 격차의 한 원인).

### 3.3 시설 "개수·거리"만 반영, "품질" 부재
- apt_facility_summary 는 최근접거리 + count_1km/3km/5km 만 보유(`batch/quarterly/recalc_summary.py:695`).
- education 넛지가 **school_zones(학군 테이블)를 사용하지 않고** 학교 "개수"만 사용 — 학군 넛지의 핵심 가치와 괴리.
- 병원 규모(응급실/소아과 여부), 공원 면적, 학원가 밀도 등 질적 차원 부재.

### 3.4 nature 넛지의 구성 타당성
- park .487 + **library .257 + pet_facility .256** — 도서관·반려시설이 '자연친화'의 51%를 차지. 산/하천/녹지/공기질 없음. §2.1 편향의 직접 원인.

### 3.5 가격 지표 스케일 문제
- price_score 는 시군구 상대평가(`recalc_price.py:174-176`) — 비싼 시군구 내 상대적 저렴함만 반영, 절대 가격 부담 미반영.
- jeonse_ratio 는 0~100 정규화 없이 원값(전세가율 %)이 가중합에 직접 주입(`nudge.py:235`) — 다른 0~100 지표와 스케일 불일치.

### 3.6 다중 넛지 단순평균
- `calculate_multi_nudge_score` scoring.py:343-351 — 선택 넛지 간 중요도 차등 없음. 사용자가 3개 선택 시 우선순위 표현 불가.

### 3.7 하드코딩 파라미터 + 근거 부재
- decay/density factor/지역 프로필 배율(scoring.py:67-107), 안전 v3 구간 점수(recalc_summary.py:209-239) 등이 검증 근거 없는 고정값. `models/distance_curves.json`(ML 산출물)은 런타임에 **로드되지 않음**.
- 지역 프로필이 시도 앞2자리 3단계(metro/major_city/provincial)로 조잡 — 같은 도내 도시/농촌 무시(scoring.py:18-25).

### 3.8 검증·피드백 루프 부재
- user_event/chat_feedback 은 수집만 되고 스코어링·ML 어디서도 소비되지 않음(grep 0건).
- train_scoring.py(XGBoost)→update_weights.py 는 batch/run.py 파이프라인에 없음 — **1회성 수동 실행 후 정체**.

### Phase 1 — 단기 — ✅ 1-1/1-3/1-4 구현 완료 (2026-07-03, 1-2 는 hedonic 으로 대체·1-5 후속) (기존 보유 데이터 활용, 1~2주 규모)
| # | 항목 | 내용 | 대응 문제 |
|---|------|------|-----------|

| 1-2 | 가중치 근거 문서화 (AHP-lite) | 넛지별 가중치를 쌍대비교로 재도출하고 근거를 문서/DB에 기록 — "근거 있는 하드코딩"으로 격상 | §3.7 |

| 1-5 | 다중 넛지 가중 선택 | 첫 번째 선택 넛지에 가중(예: 0.5/0.3/0.2) 또는 UI 슬라이더 | §3.6 |

> **Hedonic 검증 결과 (2026-07-03, `docs/analysis/hedonic-validation-latest.md`)**: 표본 21,077 아파트, within R²(시군구 고정효과) 0.5115, 16개 시설 subtype 중 11개가 |t|≥2 유의(핵심 4시설 — subway/school/assigned_elementary/mart — 모두 유의 음수, 거리가 가까울수록 가격↑).
> school~assigned_elementary r=0.92 — 가중치 조정 시 합산 축 권장, cctv/bus 양수 계수는 도심밀집 공선성.

### 검증 체계 (모든 Phase 공통)
1. **시장 검증**: 점수 ↔ ㎡당 실거래가 상관 (hedonic, 1-3)
2. **행동 검증**: 추천 노출 ↔ 클릭/상세 진입 상관 (user_event)
3. **회귀 프로브**: 본 문서 §2 의 지역×넛지 분포/중복도 프로브를 배치 테스트로 상설화 — 파라미터 변경 시 편향 회귀 감지

## 5. 권장 착수 순서

1. **Phase 0-1 (crime 테이블 버그)** — 확인된 결함, 소규모, 즉시
2. **Phase 1-3 (hedonic 검증 배치)** — 추가 데이터 없이 현행 점수의 타당성을 처음으로 "측정"하게 됨. 이후 모든 개선의 효과 측정 기반
3. **Phase 2-2 (건축물대장)** — PNU 직결로 비용 대비 차별화 효과 최대
4. nature/education 재설계 (2-4, 2-5, 1-1) — 적합성 문제가 가장 뚜렷한 두 넛지
